🧠 Skill-Kernel: Autonomous Agentic OSSkill-Kernel is a production-grade orchestration layer for "Digital Employees." It moves beyond basic LLM assistance into Autonomous Agentic Workflows using a modular "Skills" framework, the Model Context Protocol (MCP), and Supabase as a persistent state-of-record.Designed for high-reliability SDLC automation, Skill-Kernel provides the connective tissue between raw model intelligence and real-world execution.🏛 Architecture OverviewSkill-Kernel operates on a State-First principle. Unlike ephemeral chat sessions, every decision, tool call, and handoff is recorded in a centralized Postgres ledger.The Brain: Claude 3.7 Sonnet / Claude Code.The Memory: Supabase (PostgreSQL) for task persistence and execution traces.The Hands: A library of "Skills"—repeatable, atomic TypeScript/Python functions.The Nervous System: Slack & PagerDuty for human-in-the-loop governance.🤖 The Digital Employee PodSkill-Kernel orchestrates specialized agent archetypes throughout the SDLC:AgentDomainPrimary ResponsibilityArchitectPlanningRequirements ingestion and task decomposition.SREOperationsRoot-cause analysis and automated system healing.QA/EvalQualityLLM-output validation and automated regression testing.SecurityComplianceRLS policy auditing and secret detection.SupervisorGovernanceLoop detection, FinOps/Budget tracking, and PagerDuty escalation.🚀 Key Features1. The Skill RegistrySkills are not hard-coded; they are registered. This allows agents to dynamically discover capabilities at runtime using JSON schemas.Atomic: Each skill does one thing (e.g., parse_logs, trigger_deploy).Portable: Hosted as Supabase Edge Functions for zero-latency execution.2. Deep Observability (Traceability)Every agent "thought" and "action" is logged into the execution_logs table. This provides a flight-recorder for:Debugging hallucinated tool calls.Auditing security decisions.Calculating exact token-spend per feature.3. Human-in-the-Loop (HITL)Critical failures or high-budget tasks trigger a PagerDuty incident. Agents pause execution until a human provides an Approve or Redirect signal via Slack Block Kit buttons.📂 Project StructurePlaintext├── .claude/               # Agent behavior configurations

🛠 Getting Started1. Initialize the BrainEnsure you have the Claude Code extension or CLI installed.Bashclaude config set theme natural-fibers
2. Provision the StateDeploy the Supabase migrations to set up your kernel's memory:Bashsupabase migration up
3. Register a SkillAdd a new capability to the registry:SQLINSERT INTO skill_registry (skill_name, schema, handler_url)
VALUES ('slack_notifier', '{...}', 'https://xyz.supabase.co/functions/v1/slack');
🛡 Governance & SafetyIdempotency: All skills must be safe to retry.Recursive Guardrails: The Supervisor agent kills any task exceeding 5 iterations without a state change.Budgeting: Hard limits on token usage per task_id.
